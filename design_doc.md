### US0 · 新用户：完成身份问询与文化标签推荐（Onboarding）

**故事**
> 作为第一次打开 Memory Roots 的用户，我想通过几个简单问题告诉产品我是谁、来自哪里，然后立刻看到和我相关的文化标签与故事推荐，感受到「这个产品是为我准备的」。

**问询流程（3步，每步一屏）**

| 步骤 | 问题 | 输入形式 |
|------|------|---------|
| Step 1 | 「你主要是哪种角色？」 | 单选卡片：我想记录家人的故事 / 我想探索各地的文化 / 两者都有 |
| Step 2 | 「你的家乡在哪里？」 | 地区选择器（省 → 市）或手动输入 |
| Step 3 | 「你对哪类文化记忆最感兴趣？」 | 多选标签（最多 3 个）：民族服饰 · 方言与歌谣 · 传统手工艺 · 节庆习俗 · 饮食与食谱 · 家族故事 · 建筑与村落 · 信仰与仪式 |

**验收标准**

- [ ] Onboarding 共 3 屏，每屏一个问题，进度条显示当前步骤（1/3）
- [ ] Step 1 角色选择决定默认落地页：记录者 → 进入「录音」首页；探索者 → 进入「地图」首页
- [ ] Step 2 家乡信息自动将 Mapbox 地图定位至对应地区
- [ ] Step 3 所选标签提交后，AI 根据标签组合在地图上高亮相关文化标记点，并在首页推荐 3 条匹配的公开故事
- [ ] 全程可跳过，跳过后以「探索者 · 全国范围 · 全部标签」作为默认配置
- [ ] 完成后可在「设置」中随时修改偏好

**涉及能力**：`Onboarding 问询` · `角色分流` · `AI 标签推荐` · `Mapbox 定位`

### US1 · 小美与奶奶：触发记忆锚点 (Seed Context)

**故事**
> 作为记录者，我不知道该怎么让奶奶自然地开口讲过去的事情。我希望只要上传一张老照片或输入一个词，AI 就能帮我生成几个好问题，作为聊天的开场白。

**验收标准**

- [ ] 首页提供醒目的「新建记忆」入口，支持上传照片或输入简短文字（如"1990年过年"）。
- [ ] 上传后进入 Loading 态，调用 MiniMax 多模态大模型分析图片内容（场景、年代感、人物关系）。
- [ ] 页面即刻渲染出 3 个基于图片的"破冰问题"（Ice-breakers），例如："照片里左边那个小男孩是谁？"

**涉及能力**：`多模态图像理解` · `AI 破冰提问`

### US2 · 奶奶：在极简访谈模式下自然倾诉 (Multi-modal Capture)

**故事**
> 作为不擅长用手机的老人，我不需要理解复杂的界面。我只想看着屏幕上的老照片和问题，像平时聊天一样把故事说出来。

**验收标准**

- [ ] 页面进入"访谈模式"：上半部分居中展示老照片，中间大字显示 AI 生成的 1 个当前问题。
- [ ] 下半部分为极简的"按住说话"或"点击录音"按钮，辅以温暖的声波 UI 动效，无其他干扰导航。
- [ ] 长辈录音时，支持随时停止或补充录入（仅限音频，为确保体验流畅，暂不引入视频）。
- [ ] 录音结束后点击"完成讲述"，触发后台大模型处理流水线。

**涉及能力**：`极简适老化 UI` · `语音数据流采集`

### US3 · 小美：AI 实时整理与逻辑追问 (Agentic Follow-up)

**故事**
> 作为记录者，我希望 AI 帮我把奶奶（可能带有口音）的语音转成文字，提取关键信息。如果故事里漏了重要细节，我希望 AI 能提醒我马上追问，避免以后再也想不起来。

**验收标准**

- [ ] 提交录音后，系统显示"正在整理回忆..."，后台并行执行：语音转写、信息抽取（NER）、摘要生成。
- [ ] 逻辑校验与追问反转：若 AI 判断提取的数据中缺失关键节点（如提到了大事件但无时间/地点），屏幕弹出追问建议。例如："奶奶提到了'那年大雪'，要不要问问她是哪一年？"
- [ ] 小美可选择"继续追问"（长辈补充回答）或"保存结束"。
- [ ] 最终保存时，后端向前端返回标准化的结构 JSON（包含 `memory_id`, `summary`, `transcript`, 及 `time`/`location`/`people` 的 tags 对象）。

**涉及能力**：`语音识别` · `RAG 数据清洗` · `Agentic 逻辑决策` · `结构化 JSON 输出`

### US4 · 家族：在岁月时间轴中探索数字记忆卡 (Digital Relic Display)

**故事**
> 作为家族成员，我希望看到一个清晰的时间线，所有的故事都变成一张张带有年代感的卡片。我甚至希望可以直接向这段记忆提问。

**验收标准**

- [ ] 岁月时间轴 UI：放弃传统列表，采用贯穿屏幕的垂直时间轴，基于 AI 提取的 `time` 标签挂载节点。
- [ ] 记忆卡片展现：拍立得风格卡片，包含封面照片、AI 摘要、以及彩色分类标签（如：地点、情感、人物）。
- [ ] 沉浸式播放：点击卡片播放键，重现长辈原音，下方转写文字实现"卡拉OK式"同步高亮。

**涉及能力**：`响应式时间轴渲染` · `前端动态高亮`

（注：US5 授权发布 与 US6 地图探索 维持原逻辑不变，将 US4 生成的记忆卡片脱敏后映射至 Mapbox 即可）

### US5 · 小美：将故事发布到「家乡地图」

**故事**
> 作为记录者，我想选择几条故事，标注到地图上奶奶出生的村庄，让更多人看到这里曾经有过的文化。

**验收标准**

- [ ] 选择故事后，在 Mapbox 地图上点选或搜索具体地点
- [ ] 可设置可见范围：`仅家人` / `公开给所有人`
- [ ] 发布后在地图上生成「记忆标记点」，图标按类型区分（习俗 / 技艺 / 人物）
- [ ] 点击标记可预览：故事摘要 + 标签 + 播放按钮

**涉及能力**：`Mapbox 标注` · `授权发布`

### US6 · 探索者：在地图上发现家乡的故事

**故事**
> 作为对苗族文化感兴趣的陌生人，我想在地图上探索贵州，看到真实家庭留下的文化记忆点，点击收听原汁原味的故事。

**验收标准**

- [ ] Mapbox 地图聚合展示公开记忆标记（cluster 聚合，缩放后展开）
- [ ] 标记按类型区分颜色：`技艺` · `习俗` · `人物`
- [ ] 点击标记展开故事预览卡，可直接播放录音
- [ ] 支持按标签过滤（如「蜡染」「古歌」「苗族婚俗」）
- [ ] 支持按地区浏览（省 → 市 → 县 → 村）

**涉及能力**：`Mapbox 地图探索` · `文化图谱` · `公共内容发现`

## 屏幕流总览 (更新后)

```
【新用户 Onboarding】
角色问询 → 家乡选择 → 文化标签偏好
        ↓
【私人家庭记忆树 (Part 1)】 ── (若是探索者，可直接跳至 Part 2)
新建记忆 ──→ 上传照片 / 输入提示文字 (记忆锚点)
        ↓
MiniMax 视觉识别 ──→ 生成 3 个破冰问题 (Ice-breakers)
        ↓
进入极简访谈模式 ──→ 长辈看图说话 (大字 + 声波动画)
        ↓
MiniMax 后台清洗 ──→ 转写 / 翻译 / 抽取标签结构化 JSON
        ↓
信息完整度校验 ──→ [发现关键信息缺失] ──→ 弹出追问建议 ──(长辈补充回答)──↑
        ↓ (信息完整 / 用户选择直接保存)
生成「数字记忆卡片」 ──→ 挂载至「岁月时间轴」
        ↓
【全球文化指南 (Part 2)】
选择记忆卡片 ──→ 隐私脱敏 / 授权发布 ──→ Mapbox 文化地图聚合展示
```

## 技术关键点

| 模块 | 方案 |
|------|------|
| Onboarding 问询 | 3 步单页问卷，答案本地存储，完成后写入用户偏好配置 |
| 标签推荐 | Claude — 根据角色 + 家乡 + 兴趣标签组合，返回匹配的文化标记与推荐故事 |
| 录音分享页 | 无需登录的分享链接，Web Audio API，支持微信内置浏览器 |
| 语音转写 | OpenAI Whisper（支持方言识别） |
| AI 整理 | Claude — 提取标签 / 生成摘要 / 识别地名实体 / 情感分类 |
| 地图可视化 | Mapbox GL JS — 聚合标记、分类图标、故事弹窗 |
| 存储 | 音频文件 → 对象存储；结构化数据 → 关系数据库 |
| 权限控制 | 故事级别权限：私人 / 家庭 / 公开 |

## 设计原则

1. **奶奶能用** — 每一个面向长辈的界面，操作步骤不超过 1 步
2. **小美放心** — 录音、照片永久保存，随时可导出
3. **故事第一** — UI 退后，内容与情感前置
4. **渐进公开** — 默认私人，主动授权才进入公共地图
